# Text Refine

Claude 直接校准 ASR 转录文本，修正错别字、同音字、专业术语，并输出排版精美的 Markdown。非中文自动翻译为中文后校准。

**无需额外依赖** — Claude 自身就是校准引擎。

## 工作流程

1. 读取用户指定的字幕文件（SRT / MD / TXT）
2. 自动检测语言
3. 非中文 → 翻译成中文 → 校准
4. 中文 → 直接校准
5. 输出校准后的 Markdown 到同目录 `{原文件名}_refined.md`

## 首次检查

无额外依赖，跳过。

## 校准规则

### 核心原则

- **基于全文主题和语义校准**，不孤立地校对每个句子
- 同音字的选择、术语的确定都要基于上下文语境
- 先理解全文主题，再逐段校准

### 删除 Whisper 冗余词

Whisper 经常误添加无意义的"叫做"、"所谓的"等词，必须删除：
- "叫做被抑制" → "被抑制"
- "所谓叫做这个功能" → "这个功能"
- **原则**：删除后句子仍通顺就不要加回任何词

### 清理 Whisper 重复伪影

Whisper 常在段落末尾或段落之间产生整句重复，必须清理：
- "我增加了一对新婚夫妇" × 5 → 保留 1 次
- "观看观看观看观看观看" → "观看"
- **原则**：同一短语连续重复 3 次以上，保留 1 次；保留后检查语义是否通顺

### 清理平台水印和广告

从视频平台下载的视频常包含水印文字被 ASR 识别，必须删除：
- "优优独播剧场"、"YoYo Television Series Exclusive" → 整段删除
- "Extension"、"解锁更多"等平台提示语 → 删除
- **原则**：明显非视频内容本身的广告、水印、平台提示语一律删除

### 清理 Whisper 幻觉内容

Whisper 在中文音频中常幻觉出英文或无意义内容，尤其在段落边界：
- 中文视频中突然出现的无关英文短句（非引用、非专有名词）→ 删除
- 段落末尾的 "stylestyle"、"yeah yeah yeah" 等无意义重复 → 删除
- **原则**：内容与上下文语义无关且无法合理解释的，视为幻觉并删除

### 添加标点符号

原始 ASR 输出通常无标点，校准时必须补充：
- 句号（。）、逗号（，）、问号（？）、感叹号（！）、冒号（：）、分号（；）
- 作品名用书名号（《》），重要概念首次出现可用引号（""）
- 人名中间加分隔号（马丁·斯科塞斯）
- 省略号用标准中文省略号（……）
- **原则**：标点服务于可读性，不改变语义和节奏

### 同音字/错别字纠正

根据上下文语义判断，常见 ASR 错误类型：

**通用**：
- "地域" → "阈值"
- "一笔代过" → "一笔带过"
- "记录" → "纪录"（纪录片语境）
- "连接" → "链接"（网址语境）
- "涉及" → "设计"（设计语境）

**影视领域**：
- "试点" → "视点"（point of view）
- "抚痒" → "俯仰"（镜头俯仰运动）
- "更生降" → "跟升降"（镜头运动分类）
- "组合剂" → "组合技"（组合技巧）
- "类工" → "内功"（武术比喻）
- "被设主体" → "被摄主体"（被拍摄的主体）
- "推轨尽头" → "推轨镜头"
- 外文人名：马丁西克赛斯 → 马丁·斯科塞斯、格达尔 → 戈达尔、西区科科 → 希区柯克
- 外文片名：巴黎灵灯 → 巴里·林登

**科技领域**：
- "Crawl" → "Cursor"（AI 编程工具）
- "Google fly" → "Google Flights"

**医疗领域**：
- "肃部" → "腹部"
- "隔肌" → "膈肌"
- "肋股" → "肋骨"

### 不改变原意

- 不添加内容
- 专有名词和产品名称必须准确
- 不随意更改句式

## 视频类型自动检测

读取文本前 30 段样本，根据关键词判断类型并调整校准策略：

| 类型 | 关键词示例 | 校准重点 |
|------|----------|---------|
| **影视** | 镜头、导演、摄影、景别、分镜、推拉摇移、视点、调度 | 人名片名准确、影视术语准确、外文片名翻译 |
| **医疗** | 膈肌、骨盆、康复、腹内压、骶骨 | 解剖学术语、医学术语准确 |
| **科技** | 代码、API、Cursor、Python、React | 产品名称、技术术语准确 |
| **财经** | 股票、基金、K线、涨停板 | 财经术语、公司名称准确 |
| **通用** | 其他 | 常见 ASR 错误纠正 |

检测到特定类型后，校准时优先使用对应领域的术语。

## 分段处理

- 短文件（≤200 段）：一次读取，整体校准
- 长文件（>200 段）：分段处理，每段保留前段最后 5 句作为上下文衔接，确保跨段语义连贯

## 输出格式

输出 Markdown 文件，保存到输入文件同目录，文件名加 `_refined` 后缀。

### 文件结构

```markdown
# {标题}

> 📝 {主题一句话概要} · 校准于 {YYYY-MM-DD}

---

{正文内容}
```

头部保持简洁：标题 + 一行引用式摘要。不堆砌元信息段落。

### 格式美化规则

根据内容语义重建 Markdown 结构，让输出文件具备良好的可读性：

**层级标题**：
- 根据内容主题添加 `##` 级标题，将连续话题拆分为独立章节
- 章节内有子话题时用 `###` 级标题
- 标题要简洁，反映该段内容的主旨

**加粗关键术语**：
- 专业术语、核心概念在首次出现时加粗（`**术语**`）
- 后续重复出现不再加粗
- 人名、作品名不加粗（用书名号《》标记作品）

**列表化枚举**：
- 原文中的分类、类型、要点等枚举内容，转为无序或有序列表
- 每个列表项可以是短语或短句，不必是完整段落

**分段**：
- 按语义单元分段，避免大段文字（每段 3-8 句为宜）
- 话题转换时必须分段
- 原文一个长段包含多个话题时，拆分为多个段落并加标题

**引用块**：
- 重要引用、名言金句用引用块（`>`）突出
- 不是所有内容都需要引用，仅限确实重要的引用

**分隔线**：
- 主要章节之间用 `---` 分隔

### SRT 时间戳处理

如果输入是 SRT 格式（有时间戳），输出中保留时间戳信息：

```markdown
**[00:01:23,000 - 00:01:25,000]**
校准后的文本
```

### 格式美化示例

输入（ASR 原始文本）：
```
首先是固定轴心运动摄影机位置不动仅改变镜头的指向角度就是固定轴心运动典型的有基于水平角度的摇基于垂直角度的俯仰运动
```

输出（格式美化后）：
```markdown
## 一、固定轴心运动

**固定轴心运动**指摄影机位置不动，仅改变镜头的指向角度。典型的运动方式包括：

- **摇**：基于水平角度的运动
- **俯仰**：基于垂直角度的运动
```
