# Text Refine

Claude 直接校准 ASR 转录文本，修正错别字、同音字、专业术语。非中文自动翻译为中文后校准。

**无需额外依赖** — Claude 自身就是校准引擎。

## 工作流程

1. 读取用户指定的字幕文件（SRT / MD / TXT）
2. 自动检测语言
3. 非中文 → 翻译成中文 → 校准
4. 中文 → 直接校准
5. 输出校准后的 Markdown 到同目录 `{原文件名}_refined.md`

## 首次检查

无额外依赖，跳过。

## 校准规则

### 核心原则

- **基于全文主题和语义校准**，不孤立地校对每个句子
- 同音字的选择、术语的确定都要基于上下文语境
- 先理解全文主题，再逐段校准

### 删除 Whisper 冗余词

Whisper 经常误添加无意义的"叫做"、"所谓的"等词，必须删除：
- "叫做被抑制" → "被抑制"
- "所谓叫做这个功能" → "这个功能"
- **原则**：删除后句子仍通顺就不要加回任何词

### 同音字/错别字纠正

根据上下文语义判断：
- "肃部" → "腹部"
- "隔肌" → "膈肌"
- "肋股" → "肋骨"
- "地域" → "阈值"
- "Crawl" → "Cursor"（AI 编程工具）
- "Google fly" → "Google Flights"

### 不改变原意

- 不添加内容
- 专有名词和产品名称必须准确
- 不随意更改句式

## 视频类型自动检测

读取文本前 30 段样本，根据关键词判断类型并调整校准策略：

| 类型 | 关键词示例 | 校准重点 |
|------|----------|---------|
| **医疗** | 膈肌、骨盆、康复、腹内压、骶骨 | 解剖学术语、医学术语准确 |
| **科技** | 代码、API、Cursor、Python、React | 产品名称、技术术语准确 |
| **财经** | 股票、基金、K线、涨停板 | 财经术语、公司名称准确 |
| **通用** | 其他 | 常见 ASR 错误纠正 |

检测到特定类型后，校准时优先使用对应领域的术语。

## 分段处理

- 短文件（≤200 段）：一次读取，整体校准
- 长文件（>200 段）：分段处理，每段保留前段最后 5 句作为上下文衔接，确保跨段语义连贯

## 输出格式

输出 Markdown 文件，保存到输入文件同目录，文件名加 `_refined` 后缀：

```markdown
# {标题}

**Refined by**: Claude
**Refined at**: {YYYY-MM-DD HH:MM:SS}
**Segments**: {段落数}
**Context**: {主题一句话概要}

**校准状态**: ✅ 已通过 Claude 校准

{校准后内容，每段一行，段落之间空行分隔}
```

如果输入是 SRT 格式（有时间戳），输出中保留时间戳信息：

```markdown
**[00:01:23,000 - 00:01:25,000]**
校准后的文本
```
