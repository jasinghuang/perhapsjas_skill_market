# 修复政策（最小切片）

按**最小单位（通常是单张卡）**修复。**有修复才写** `review/repair-log.md`（一次过 / 无修复则不写）。

## 禁止

- 用户只反馈一处问题就**重写全套卡片**。
- 为了修视觉而改动 Phase 3 已确认的**拆分结构**（卡数 / 边界）。
- 为了压缩信息而删除用户**指定必须保留**的内容。
- 为修一张卡而改动别的卡（除非问题本身就是跨卡衔接）。

## 合规红线（金融投教场景 · 不可越界）

风险提示是布局模板自带的**固定文案**，`position: absolute` 钉底每张卡片底部：

- 任何修复都**不得修改、删减、移动、替换**风险提示文案。
- 若某卡风险提示丢失或损坏，**只能从对应布局模板原样恢复**，不得改写。
- 这条优先级高于一切排版 / 视觉考虑——违反 = 合规事故，不是 bug。

## 最小切片对照

| 问题 | 最小修复单位 |
|---|---|
| 单卡内容缺失 / 错误 | 对应 `cards/card-NN.html` |
| 单卡溢出 / 排版 | 对应 `card-NN.html`（调整该卡内容块 / 关键词标记，不动别的卡） |
| 关键词标记过度 / 遗漏 | 对应 `card-NN.html` 的 `<span class="highlight">` |
| 风险提示问题 | 对应布局模板的 disclaimer（**只恢复固定文案，不改写**，见合规红线） |
| 拆分不对（卡数 / 边界） | 回到 Phase 2 改 `plan/plan.md` Outline + 重做受影响的卡（不是全套） |
| 封面主题词 / 副标题 | `cards/card-01.html` + `plan/plan.md` Brief 段 |
| 全套风格不统一 | 逐卡修（每张 `card-NN.html` 单独改），不重写 |
| 卡片序号错乱 | `cards/manifest.json` + 受影响卡的文件名 |
| HTML 打不开 / 样式塌陷 | 对应 `card-NN.html` 的具体结构 |

## repair-log.md 格式

```
## <日期> <谁反馈 / 哪个 Reviewer>
- 问题：<一句话>
- 定位层：<内容 / 排版 / 结构 / 合规 / 构建>
- 最小修复单位：<card-03 / manifest.json / plan.md Outline ...>
- 改动：<改了什么>
- 验证：<open 预览 / Card Reviewer 复核通过>
```

先定位是哪一层（内容 / 排版 / 结构 / 合规 / 构建），再改最小切片，**不要重做全套**。
