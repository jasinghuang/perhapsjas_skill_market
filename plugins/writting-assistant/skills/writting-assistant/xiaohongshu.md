# 小红书金融投教自媒体文案创作

本 Skill 旨在指导你完成高质量自媒体文案的全自动创作流程。

---

## 核心工作流

### 1. 初始化与需求分析

**确定主题**

- 用户想写什么？

**分析输入**

- **参考资料**（可选）：作为信息来源
- **参考文案**（可选）：作为风格/结构模仿对象（爆款对标）

**确定分类**

- 检查 `examples/` 下的子文件夹
- 判断本次写作属于哪个或哪些分类（例如 `examples/tech/`、`examples/life/`）
- ⚠️ **关键步骤**：检查该分类目录下是否存在 `standards.md`（例如 `examples/tech/standards.md`）。如果存在，**必须**读取并遵守其中的所有规定（包括文风、是否联网、是否找素材、是否需要素材等），后续所有阶段均需检查此配置

---

### 2. 资料研究阶段（Deep Research 循环）

> **目标**：确保有足够的信息来撰写文案

**检查配置**

- 读取分类的 `standards.md`。如果其中注明"无需联网"、"跳过搜索"或类似指令，则**直接跳过**本阶段

**评估现状**

- 当前的参考资料是否充足？

**搜索循环**

- 如果资料不足，根据主题和现有资料生成搜索关键词
- 使用 `WebSearch` 同时查找 **中文** 和 **英文** 来源
- **筛选**：优先采用最新资料。注意区分相似但过时的信息，以最新发布的为准
- **循环**：重复搜索直到信息量满足写作需求。重复搜索的时候换用关键词，像树形结构一样逐步深入不同的角度、维度

---

### 3. 风格与结构习得

**加载人设**

- 读取 `references/personality.md`，确立基础人设与语气

**加载参考样例**

- 在 `examples/{分类}/` 中寻找最接近的爆款文案作为风格参考
- 如果没有完全匹配的，参考相近分类
- **模仿原则**：参考文案越相似，结构模仿越深入；如果相似度低，仅模仿语气和节奏

**提取爆款元素**

- 分析参考文案的"开头（黄金3秒）"、"结尾（CTA）"、"情绪节奏"等核心元素
- 在不抄袭内容的前提下，将这些**爆款逻辑**迁移到新文案中

---

### 4. 文案撰写

生成初稿，文稿必须符合"5. 质量保证循环"中的标准，且必须包含以下部分（除非 `standards.md` 另有要求）：

**封面标题**

- 吸引眼球，提高点击率，文字极简

**文字标题**

- 不超过 20 字，吸引眼球，提高点击率，有可能的话尽量 SEO 优化，包含重要关键词

**正文描述**（Description）

- 30-50 字左右，尽量 SEO 优化，包含重要关键词
- 不要剧透最核心的方法、剧情、故事结局等
- 最后包含 3-4 个 Tag，例如 #AI、#谷歌、#提示词、#智能体 等等

**具体的正文**（Script/Body）

---

### 5. 质量检测循环（QA Loop）

**加载标准**

- **全局标准**：`references/quality_check.md`
- **分类独立标准**：`examples/<分类>/standards.md`（优先级更高）
- 对应的全局文件： `references/quality_check.md`

**自我检查**

- 是否符合人设？
- 是否符合爆款逻辑？
- 是否满足子目录中 `standards.md` 中的特殊要求？

**修正**

- 如果测试不通过，修改并重新检查，直到符合标准

---

### 6. 存档

使用 `scripts/save_draft.py` 脚本保存最终内容

**路径格式**

- `口播文案/YYYY-MM/XX-标题.md`

**操作指令**

```bash
# 先将内容写入临时文件，临时文件名必须带上时间戳以防止冲突（例如：temp_content_202008081245.txt）
# （必须先使用 Write 工具创建 temp_content_<timestamp>.txt）

python3 xiaohongshu_content/scripts/save_draft.py "中文极简关键词标题" "temp_content_<timestamp>.txt"
```

**注意**

- 文件名中的标题应为"中文极简关键词"（xx-xxxx）

---

### **7. 人工校验**

- 向用户展示保存的文件路径
- **暂停**：等待用户确认。用户可能会提出修改意见，此时需返回修改并重新存档（回到阶段4或5）

---

### **8. 素材搜集（Post-Approval）**

**触发条件**

1. 用户明确确认文案没问题（阶段7通过）
2. 检查分类 `standards.md`：
  - 如果注明"无需素材"，则**不主动询问**，除非用户强制要求
  - 否则，询问用户是否需要素材

**执行**（如果需要）

- 针对文案中的关键画面，生成搜索关键词
- 使用 `WebSearch` 搜索相关网页
- 使用 `WebFetch` 解析网页 HTML，提取图片或视频链接
- 下载所有可能的素材，并保存在文案相同目录
- 对于无法整理或需要用户自行收集的素材，将素材清单整理给用户，并保存在文案的 md 文件的末尾

---

### **9. 保存或更新到 Feishu**

1. 当需要生成 Feishu 文档时，页面保存到 `罐罐文案\年份名称（如2026）\月份名称（如1月）`
2. 标题和 markdown 文件名相同但去除后缀
3. 上传的时候，使用 feishu cli工具实现。相关 Token（API Key）已经配置好
