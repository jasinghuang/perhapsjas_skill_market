# 质量检测标准

> 适用于所有分类的全局质量检测清单。各分类的 `standards.md` 可覆盖或补充本文件。

## 一、合规性检查

- [ ] 未包含具体股票/基金推荐代码
- [ ] 未承诺或暗示具体收益
- [ ] 涉及投资相关内容时，包含风险提示
- [ ] 未使用绝对化用语（"一定赚"、"稳赚不赔"等）
- [ ] 内容不构成投资建议的表述

## 二、标题质量

- [ ] 封面标题足够吸引眼球，文字极简
- [ ] 文字标题不超过 20 字
- [ ] 文字标题包含重要关键词（SEO 优化）
- [ ] 标题不剧透核心内容

## 三、正文描述（Description）

- [ ] 30-50 字左右
- [ ] 包含重要关键词（SEO 优化）
- [ ] 未剧透最核心的方法/结局
- [ ] 包含 3-4 个相关 Tag（#标签格式）

## 四、正文内容

- [ ] 开头有"黄金3秒"钩子（引起好奇心或共鸣）
- [ ] 结尾有明确的 CTA（引导互动：点赞/收藏/评论/关注）
- [ ] 段落简短，适合手机阅读
- [ ] 逻辑清晰，有完整的信息链
- [ ] 无错别字、语病
- [ ] 数字、数据准确无误（如有引用需可追溯）

## 五、人设一致性

- [ ] 语气风格符合 `personality.md` 的设定
- [ ] 自称方式一致
- [ ] 专业程度与目标受众匹配

## 六、爆款逻辑

- [ ] 有情绪起伏（非平铺直叙）
- [ ] 有痛点/痒点/爽点中的至少一个
- [ ] 提供了可感知的价值（信息增量/情绪价值/实用工具）
- [ ] 内容有"转发动力"（值得分享给他人）

## 七、去AI味检测

> QA Loop 全部通过后独立执行。命中任一规则即生成问题报告，全部通过则输出"AI味检测通过，无需修改。"

| # | 规则名 | 检测内容 | 命中动作 |
|---|--------|----------|----------|
| 1 | 营销大词 | 赋能、闭环、打通、底层逻辑、抓手、矩阵、链路、护城河、势能、降本增效 | 删除或替换为平实表达 |
| 2 | 替读者说话 | "你可能觉得……""很多人会问……""你一定也遇到过……""大家都在想……" | 删除，改为直接陈述观点 |
| 3 | 指令语气 | "记住""一定要""核心就一句话""划重点""注意了""敲黑板" | 替换为平等对话语气 |
| 4 | 虚构数据 | "90%的人不知道""99%的人都做错了""大部分人""无数人"等无来源的百分比或泛指 | 删除数字或补充真实来源 |
| 5 | 独立短句装戏剧感 | 连续出现独句成段（一句话单独成行，前后有句号结尾的孤立短句≥2处） | 合并为正常段落 |
| 6 | 加粗口号金句 | **真正厉害的人……** **优秀的人都在……** **成功的关键就是……** 等加粗总结句 | 去掉加粗，改为自然融入正文 |
| 7 | 宣传腔 | 颠覆性、划时代、游戏规则改变者、革命性、史诗级、震撼登场、震撼发布 | 替换为平实描述 |
| 8 | 硬拔高 | 关键时刻、重要里程碑、历史性、标志性时刻、深远影响、意义非凡 | 删除或降低调门 |
| 9 | 无来源模糊归因 | "专家认为""研究表明""有数据显示""众所周知""业内人士指出"等无具体来源的引用 | 删除或补充具体来源 |
| 10 | 口号式短句 | 连续出现句式相同的短句（≥3句结构一致，像喊口号） | 合并或改写为正常段落 |
| 11 | 过度破折号 | 单篇使用破折号（——）超过 3 次 | 减少至 1-2 次，多余改用逗号或句号 |
| 12 | 过度排比 | "不是…而是…"结构超过 2 组，或其他排比句超过 2 组 | 精简排比，至多保留 1 组 |
| 13 | 空洞完整 | 每句话都正确但缺少具体事实/数据/案例，看起来完整但没有一句真实的话 | 补充真实细节或删除空话 |

**命中时输出格式：**

```markdown
### AI味检测报告

| 原文 | 规则 | 建议 |
|------|------|------|
| "底层逻辑其实就是…" | 营销大词 | 删除"底层逻辑"，直接说"原因是…" |
```

**未命中时输出：** `AI味检测通过，无需修改。`

## 八、内容价值审核

> 检查文案是否提供可感知的具体价值，而非"看起来有用但实际无法操作"。

- [ ] 是否包含至少一个具体案例或真实经历（非泛泛而谈）
- [ ] 是否有值得收藏的结构（清单/框架/步骤/对比表，读者会想存下来）
- [ ] 是否有一句话可以被截图（观点鲜明、表达精炼，适合单独传播）
- [ ] 是否有一句话会让人想转发（触发共鸣或认知刷新）
- [ ] 是否帮读者省下了未来一次工作（给出可直接复用的模板/清单/话术/框架）
- [ ] 是否有数据、截图或真实案例支撑（而非纯观点输出）
- [ ] 没有作者在旁边解释，读者能否直接上手使用
- [ ] 是否有适合截图或视觉化的部分（表格/对比/清单等结构化内容）

## 九、读者定位审核

- [ ] 目标读者是否具体（不是"所有人"，而是明确的人群画像）
- [ ] 使用场景是否具体（读者在什么情境下会看到这篇、为什么会看）
- [ ] 文案假设的读者知识水平是否合理（不过高也不过低）
