# 技术分类标准

> 适用于 AI 工具、编程、科技产品、效率工具、技术趋势、副业方法论等相关主题
> 框架来源：finance-article-deconstruction-card.md
> 基于 10 篇示例文章的跨文章模式分析（鱼皮 x1、Sac x1、Kafka x3、Russell3402 x1、大神王 x3、ai_xiaomu x1）

---

## 基础信息

- **主题领域**：AI 工具、编程实践、科技产品、效率工具、技术趋势、副业方法论、内容创作
- **文章类型**：攻略教程型占绝对主流（50%），其次框架拆解型（20%）和观点评论型（20%），清单式（10%）
- **联网搜索**：是。技术类内容更新快，需查最新工具版本和行业动态
- **素材需求**：教程类必须有截图/配图/代码，分析类可选

---

## 核心观点与论证

- **核心观点表达方式**（按实际频次排序）：
  1. 框架重新定义（50%）：创造新概念框架来组织读者认知——"补贴付费期→收割期""薄壳公司""编排者经济""PPT as Code五要素"
  2. 反共识先声夺人（40%）："你以为你是客户，你是数据、是入口、是燃料""大多数人以为的爆款规律，至少有一半是错的"
  3. 实战案例证明（40%）：鱼皮用"40亿tokens"建立可信度；ai_xiaomu用收入公式和真实案例数字
  4. 数据推翻常识（20%）：Kafka的"1441条样本统计分析"
  5. 故事引出洞察（20%）：两个朋友的故事、具体人物命运对比
- **"你以为X，其实是Y"结构**是科技类文章的标志性表达——所有10篇文章都包含至少一次认知纠偏
- **核心观点表达时机**：
  - 教程型：痛点对比后引出（"以前要3小时，现在10分钟"）
  - 观点型：反常识现象开场，层层追问本质
  - 框架型："大多数人的认知是错的"开场，逐一推翻后给新框架
- **用于论证的工具**：
  - 个人实战数据（50%）：具体消费量/用户量/收入数字——"40亿tokens""2000个站群网站""单品5000+份 x 39.9元 = 20万"
  - 逻辑推演链条（40%）："如果A→那么B→而B意味着C→所以你应该D"
  - 具体产品/工具对比（40%）：并列呈现A和B各是多少，不要只说A好
  - 历史类比（30%）：千团大战→网约车补贴类比AI的补贴周期；蒸汽机/电力/互联网的财富集中史
  - 可操作prompt/代码（40%）：教程型必备，给出可复制的具体指令

## 叙事结构脉络

三种结构模板，按文章类型选择：

### 攻略教程型（最主流，50%）
```
痛点共鸣开场 → 可信度锚点（"我做到了"）→ 分步骤教程（显式编号）→ 避坑/局限性说明 → 行动号召
```
- 开场钩子：直接点出读者正在经历的痛点（"AI做出来的PPT太丑""打开Claude Code，对话框空空的"）
- 可信度锚点：放在开头，用个人数据或成果
- 每一步都给出可操作的指令或prompt

### 观点评论型
```
反常识断言开场 → 历史类比/数据框架 → 层层递进论证（"第一层/第二层/第三层"）→ 预测/判断 → 窗口期紧迫感收尾
```
- 论证用"撕开N层"结构组织
- 结尾不是安慰，而是"窗口在收窄，你要行动"

### 框架拆解型
```
"大多数人的认知是错的"开场 → 数据/案例逐一推翻 → 新框架替代 → 可操作结论
```
- 每条都是"你以为X→数据/证据说Y"的结构

### 共性结构特征
- **所有文章都有"认知纠偏"环节**：即使纯教程型，也先纠正一个常见误解再进入正题
- **"先说不行/不推荐，再说怎么做"**是高频模式
- **层次标记词极其显式**："第一层/第二层""阶段一/阶段二""技巧1/技巧2""赛道一/赛道二"——不要用隐含的段落过渡
- **结尾收束**：行动号召（"现在就试试"）或紧迫感+具体方向（"窗口在收窄"），不要温情安慰

## 人设气质档案

六位作者共享一个底色——**"做了再说"的实战派**。所有文章无一例外都强调"我做出来了，不是纸上谈兵"。

### 三种可切换的人设子类型
| 方向 | 代表 | 核心气质 | 适用文章类型 |
|------|------|---------|------------|
| 分析师型 | Kafka | 冷静、数据驱动、层层拆解、解构力极强 | 趋势分析、行业洞察 |
| 顾问型 | 大神王 | 直接、坦率、有压迫感、偶尔温情、明确判断 | 转型指南、战略建议 |
| 实操者型 | 鱼皮/Sac/Russell3402/ai_xiaomu | 朴实、步骤清晰、分享具体做法 | 工具教程、工作流分享、副业攻略 |

### 共性特征
- **"我知道你不知道的事"的认知上位者姿态**：过来人或信息优势者的口吻
- 不装逼但有态度：克制炫耀，但明确表达立场
- 必须用第一人称——科技类文章不用第三人称或全知视角
- "我"的使用频率要控制：建立可信度时用（"我做了X"），论证时减少（让数据和逻辑说话）

- **专业痕迹**：能自然混用技术术语和日常用语，不刻意也不回避
- **实战痕迹**：有具体的使用截图、操作细节、真实效果数据
- **与读者的关系体感**：平视对话——战友或同行者，不端着不教你
- **克制与坦诚**：说清楚工具的局限性和成本，不要只讲好的

## 阅读氛围体感

三种情绪曲线，按文章类型选择：

### 曲线A：焦虑唤醒 → 理性安抚 → 行动赋能（观点型）
```
认知冲击（"你掏$200，对方还在亏本卖给你"）→ 框架和逻辑让读者觉得"原来如此" → 给出可操作方向
```
读者安全感："虽然情况严峻，但我知道了该怎么做"

### 曲线B：痛点共鸣 → 能力展示 → 复制路径（教程型）
```
读者正在经历的困难 → "我解决了"（建立信任）→ 可复制的步骤（降低行动门槛）
```
读者安全感："有人做出来了，而且步骤很清楚"

### 曲线C：认知颠覆 → 数据证据 → 重新校准（框架型）
```
"你之前的认知是错的"（动摇信心）→ 数据逐一证明（重建信任）→ 新框架替代（新安全感）
```

- **整体情绪关键词**：务实、高效、被启发、想动手
- **情绪触发器**：具体数字（"40亿tokens""1441条样本"）、时间紧迫感（"窗口期18-36个月"）、"大多数人都错了"的优越感暗示
- **说服力来源**：实操截图/代码 > 数据分析 > 独立观点的犀利度
- **认知节奏**：高信息密度，但用并列结构和显式编号降低认知负荷（读者可以跳读）
- **信任建立方式**：展示真实操作截图/代码 > 量化效果（"提效70%"）> 承认工具局限
- **读者的"安全感"**：行动清晰——"我现在就能试"

## 案例/数据生命点

- **核心案例的画面感**：工具使用前后的对比截图、具体操作步骤还原、两个朋友的不同命运
- **数据讲述方式**：用"你以为→数据/证据说"的结构制造认知反差。对比数据必须并列呈现
- **最具冲击力的对比**：效率对比（"3小时 vs 10分钟"）、两个朋友的命运分叉、老板A vs 老板B的ROI对比
- **案例与结论的连接方式**：演绎推理（从工具特性推到使用效果）+ 归纳总结（多个案例指向同一方法论）

## 市场语境适配

- **写作时须判断当前技术热度**：新工具发布初期 / 快速普及期 / 成熟稳定期
- **文章与热度的关系**：
  - 发布初期：信息差优势最大，快速出教程/攻略
  - 快速普及期：趋势分析型适合，讨论行业影响
  - 成熟稳定期：方法论提炼、数据研究型适合
- **时效性处理**：工具版本号、价格、功能必须标注时点。方法论类长期有效

## 写作技术痕迹

- **标志性句式**：
  - "Programmer 的价值会越来越低。Builder 的价值会越来越高。"（对比型）
  - "你掏$200，对方还在补贴你。这是一个让人需要停三秒钟才能消化的事实。"（反常识型）
  - "每一个时代的财富迁徙，都只发生在很短的窗口里。"（时间框架型）
  - "AI转型的本质只有一件事：用更低的成本，做出质量不低于原来的东西。"（定义型）
  - "行动比完美更重要。先开个店，发几篇笔记，有问题再优化。"（行动号召型）
- **过渡手法**：教程型用步骤编号自然过渡；分析型用追问推进（"为什么？因为……"）；框架型用"你以为→数据说"结构
- **金句分布规律**：
  - 观点评论型金句密度最高，平均每500字一个
  - 高频出现在：文章开头3段内、每个论证层次转折处、文章结尾最后3段
- **金句特征**：
  - 来源类型：反直觉洞察、实战经验提炼
  - 结构特点：**短句 + 对比结构 + 判断性语气**。最典型结构是"A不是B，是C"（否定+重新定义）

## 风险表达方式

三种处理模式，按文章类型选择：

### 模式1：坦率告知型（观点型/分析型文章）
- 语气直接，不回避负面——"降本最直接的路径，是裁员。这句话很难听，但是真的。"
- 先说残酷事实，再用理性框架解释为什么，降低读者情绪反弹

### 模式2：实事求是型（教程型文章）
- 指出工具/方法的局限性，但不夸大——"图文创作这块能提效的程度因内容类型差异很大""商务路演、毕业答辩，传统PPT是不能被替代的"
- 在教程中段插入"哪里不好使"的说明，增加可信度

### 模式3：分层风险标注型（副业/攻略型文章）
- 用风险等级标注——"低风险：0成本启动""中风险：版权问题""可控风险：同质化竞争"
- 结构化风险分类，降低读者的决策恐惧

### 共性特征
- **所有文章都有"坦率时刻"**：必须承认某些局限或负面，否则被读者判定为吹牛
- **"免责但不是推卸"的平衡**："我不想假装这篇文章能解决那个问题……但它能做的是：让你知道，有一个窗口，现在是开着的。"

## 给AI的创作方向

- **语气指引**：冷静的分析师而非热情的推销员。"我发现了好东西，告诉你"而不是"这个太棒了你一定要试"。允许坦率的负面表达。科技类读者不接受空洞鼓励（"你也可以的！""加油！"），要的是"这是具体怎么做"
- **结构指引**：
  - 通用结构模板：痛点/反常识开场（1-3段）→ 可信度锚点 → 认知纠偏 → 主体内容（显式编号）→ 风险/局限性说明 → 具体可操作的下一步
  - 每篇文章必须有一个"认知纠偏"环节
  - 论证层次必须用显式标记
  - 数据引用格式：必须给来源、给具体数字、对比数据并列呈现
- **人设指引**：
  - 核心人设："做了再说"的实战派。假设"我做过这些事，现在把经验分享出来"
  - 三种子类型可按文章类型切换（分析师/顾问/实操者）
  - 不能写没有具体细节的案例——"三个月后单月净利润200万"比"取得了很好的效果"有力100倍
- **专业度校准**：
  - 技术术语必须出现但附带解释：第一次出现时解释，之后直接使用
  - 具体工具名称和版本必须准确——说"Claude Code"不说"AI编程工具"
  - 引用行业数据时标注来源年份
- **需要避免的雷区**：
  - 夸大 AI 能力（"AI 能完全替代人类"）
  - 写没有证据支撑的极端断言——断言必须紧跟数据或案例
  - 教程中省略前置条件和失败场景
  - 在教程中使用模糊的成功承诺（跳过风险直接给时间线）
  - 用"大家都知道"式开头——开场必须是"你不知道的事"
  - 省略"为什么"的推理链条——科技类读者不接受"这样做就对了"的断言
  - 把不同文章类型的结构混用
  - 在观点文章结尾给"一切都会好的"温情安慰
  - 引用未经授权的付费内容
  - 用"彻底颠覆""完全取代"等绝对化表述
