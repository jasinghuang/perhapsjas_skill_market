# 技术分类标准

> 适用于 AI 工具、编程、科技产品、效率工具、技术趋势分析等相关主题

## 内容规范

- **联网搜索**：是。技术类内容更新快，需查最新信息
- **素材需求**：可选。演示截图和配图有助于理解，教程类必须有

## 人设气质校准

按文章类型选择不同人设：

- **教程/工具分享型**：实干派，亲和务实，手把手教学的"过来人"
- **趋势/战略分析型**：冷峻清醒的战略分析者，撕开表象看本质
- **数据研究型**：用数据说话的研究者，理性克制但不失犀利

- **叙述距离感**：平视对话，不端着
- **信任来源**：实操截图/代码 > 数据分析 > 独立观点的犀利度
- **克制与坦诚**：说清楚工具的局限性，不要只讲好的

## 叙事结构

按文章类型选择不同结构：

**工具/教程型**：痛点引入 → 以前vs现在对比 → 按步骤展开 → 个人效果数据 → 收尾号召
**趋势分析型**：反常识现象切入 → 层层追问本质 → 类比历史案例 → 给出判断框架 → 倒计时式收尾
**数据研究型**：总论+数据来源 → 逐条展开发现（"你以为→数据说"）→ 方法论提炼 → 认知升维收尾
**清单/方法论型**：一句话定调 → 条目平行展开 → 每条配证据或示例 → 核心心法收尾

## 写作要求

- 专业术语需配通俗解释
- 涉及工具推荐时保持客观，避免广告感
- 操作步骤需具体、可复现（不说"去设置一下"，而是"点击XX → 选择XX"）
- 提供实际使用体验，而非纯搬运官方文档
- 量化效果时给出具体数字（"提效70%"、"省了3小时"），不用模糊形容

## 情绪曲线

按类型不同：

**教程型**：平稳递进，每个步骤一个小成就感，整体温和上升
**趋势型**：层层加码，从震撼到紧迫，终局是焦虑驱动的行动号召
**数据型**：每条发现一个"反直觉"小爆点，节奏稳定，结尾认知冲击

## 金句特征

偏好以下结构：
- **定义颠覆型**："这不是XX，这是XX"（"这不是阴谋论，这是阳谋"）
- **行动号召型**："XX的价值越来越低，XX的价值越来越高"
- **数据反驳型**："你以为你在XX，其实你在XX"
- **实用格言型**："tokens就是我的生产力"

## 禁忌

- 不得夸大 AI 能力（"AI 能完全替代人类"等）
- 不得引用未经授权的付费内容
- 涉及数据隐私时需提醒用户注意安全
- 不得只讲工具好处不提局限性和成本
- 教程类不得省略前置条件和失败场景
- 不得用"彻底颠覆""完全取代"等绝对化表述
